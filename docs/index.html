Architecture

Decentralized by design.
Zero infrastructure.

Every device is both a client and a relay node. No central server, no cloud dependency.

01  BLE Peripheral + Central

Each device simultaneously acts as a Bluetooth LE central (discovers others) and peripheral (advertises itself), enabling full-duplex mesh communication.

02  Service Discovery

MeshLink devices are identified by a unique BLE service UUID. Non-MeshLink connections are automatically disconnected after service verification.

03  Chunked Transport

Large payloads are split into 160-byte base64 chunks with sequence-based reassembly. A 10-second timeout handles lost chunks gracefully.

04  State Restoration

CoreBluetooth state restoration IDs ensure connections persist across app restarts and background transitions on iOS.

Wire Protocol

Protocol v3—lean & extensible.

A minimal JSON wire format supports messages, images, typing indicators, ACKs, and mesh relay with a single unified schema.

WireMessage.swift
struct WireMessage: Codable { let v: Int // 3 let type: String // "msg" | "img" | "typing" | "ack" let id: String // UUID let sender: String // node name let text: String? // AES ciphertext let ackId: String? // delivery receipt let ttl: Int? // mesh hop count let originId: String? // original sender let imgData: String? // base64 image let imgThumb: String? // base64 thumbnail }
1

Message Flow

Messages are JSON-encoded, AES-256-GCM encrypted, then broadcast to all connected peers and subscribed centrals simultaneously.

2

Mesh Relay

Received messages with TTL > 0 are re-broadcast with decremented TTL. A seen-message cache (500 IDs) prevents loops.

3

Delivery ACKs

Receivers send back type: "ack" with the original message ID, giving senders confirmed delivery status.

4

Image Chunking

Images over 180 bytes use ChunkEnvelope with sequence numbers for ordered reassembly across BLE's limited MTU.

Security Model

Built for zero trust.

Every layer is designed assuming the network is hostile. Encryption is not optional—it's the foundation.

🔐 Key Derivation

Shared secrets are derived using PBKDF2 with HMAC-SHA256 and 100,000 iterations, producing a 256-bit symmetric key from any passphrase.

PBKDF2-SHA256

🛡️ Authenticated Encryption

All messages use AES-256-GCM, providing both confidentiality and integrity. Tampered ciphertext is detected and rejected.

AES-256-GCM

📡 Zero Network Trust

No servers, no internet, no DNS, no TLS certificates. Communication is direct device-to-device over Bluetooth LE with no intermediary infrastructure.

AIR-GAPPED

🔍 Key Exchange

Keys are shared out-of-band via NFC tap, QR code scan, or direct entry. No key exchange over the wireless channel eliminates MITM vectors.
